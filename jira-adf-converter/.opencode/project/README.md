# OpenCode Project Workflow

This folder is the canonical project memory for OpenCode work in this repository.

Read these files as needed:

- `context.md`: product purpose, architecture, entrypoints, and source-of-truth rules.
- `workflow.md`: practical steps for inspecting, changing, and verifying code.
- `testing.md`: test commands, conventions, and fixture expectations.
- `status.md`: current verified state, recent decisions, and known limitations.

The older `.roo/**` files are historical RooCode memory. Do not treat them as canonical when they conflict with source, tests, or this folder.
